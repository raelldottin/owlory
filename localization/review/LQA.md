## Checks performed

1. **format-specifier-parity** — `%@`/`%d`/`%1$@`/`%d%%` specifiers must match English in count and type. Mismatch is auto-fixed by reverting that entry to the English source.
2. **empty-value** — non-keep-english-term entries must not be empty.
3. **identical-to-english-outside-keep** — entry matches English but is not labeled `keep-english-term`. Warning only.
4. **length-outlier** — translation/English character-length ratio outside `[0.4, 3.0]` (or `[0.25, 3.0]` for CJK). Warning only.
5. **mojibake-suspect** — translation contains literal `\xNN` bytes or U+FFFD. Warning only.

## What `lqa.status=passed` does NOT mean

- It does NOT mean a native or fluent reviewer accepted the translation.
- It does NOT mean translation quality is proven for that entry.
- It does NOT change `provenance.native_reviewed`; native-review state comes only from the per-locale return file and per-entry `native_review` metadata.

New post-review entries can remain pending even when a locale return file has earlier native/fluent review provenance.
