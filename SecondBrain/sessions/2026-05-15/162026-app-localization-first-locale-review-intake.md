## Translation Method

Wrote a Python script at `/tmp/owlory_de_translations.py` that:

1. Holds a 305-entry table mapping each English source value to a German LLM-draft. Format specifiers (`%d`, `%@`, `%1$@`, `%d%%`, etc.) preserved exactly. Em-dash (`—`) preserved as the literal U+2014 character.
2. Rewrites `de.lproj/Localizable.strings` line-by-line via a strict `"KEY" = "VALUE";` regex, swapping each value if present in the table.
3. Rewrites `de.lproj/Localizable.stringsdict` via an old-style-plist state machine that tracks `current_key` + `current_var` and swaps `one`/`other` values when the (key, var, category) triple is in the stringsdict table.
4. Emits the review return file by re-parsing the current `en.lproj` resources, looking up each entry in the translation tables, and recording status + reviewer notes. Entries added to en.lproj after the original packet was generated are flagged with `post_packet_addition: true` and have a `[Added after original packet]` prefix on `reviewer_notes`.

Total replacements: 314 strings + 42 stringsdict-categories = 356 entries.

The packet was a 2026-05-13 snapshot with 320 entries; current resources have 356 entries because newer keys (readiness check-in summary lines, readiness axis/summary tiers, weekly digest insight/highlight sentences, accessibility skip/complete/restore strings, recurrence-interval plurals) shipped after the packet was generated. All of them were translated and recorded in the return file with the `post_packet_addition` marker.

## Translation Choices Worth Calling Out

- Tab labels: `Today→Heute`, `Train→Training`, `Write→Schreiben`, `Career→Karriere`, `Home→Haushalt`. `Home` is the most contested choice — `Zuhause` is more colloquial; `Haushalt` (household) better matches the Owlory contract that this tab is for recurring household work.
- Voice/formality: formal `Sie`-form used throughout per German app convention. A native reviewer may prefer informal `du` (Apple defaults vary by app).
- Brand/format/loanwords kept English (`OK`, `URL`, `Build`, `Podcast`, `Video`, `Check-in`, raw format specifiers). Status `keep-english-term` for those.
- Em-dash preserved verbatim from English notification bodies.
- Streak compact "%dd" became `%dT` (T = Tag). Recurrence-interval compact `Every %dd` became `Alle %dT`.
- Some sentences hold a `%@` substitution that may need German argument reordering (e.g., `Status: %@`, `Add %@`). The LLM-draft keeps English argument order. A native reviewer should confirm word order is natural.

## What This Proves vs. What It Does Not

Proves:

- The German `.strings` and `.stringsdict` files parse and link without error (xcodebuild succeeded).
- Localization parity is preserved: 314 keys and 13 plural keys across all 19 locales (matches origin/main parity counts).
- Every reviewed entry carries reviewer metadata. The slice's entry condition is satisfied.
- iOS users with German set will see LLM-drafted German strings at runtime instead of English placeholders.

Does NOT prove:

- That the German translations are correct, idiomatic, or natural. They are LLM-drafts.
- That the German values render correctly under all UI surfaces / Dynamic Type sizes / RTL-CJK mirroring scenarios. The Batch 7 layout regression asserts shell hittability under `-AppleLocale de_DE`, not translated-text layout.
- That argument order, gender, formality, or pluralization is correct in German. A native reviewer is still required.
- Screenshot, device, or TestFlight evidence for German content.

## Lane Boundary

`build-tested` per the slice spec. The xcodebuild compile-and-link step is the gating proof. Native-review, running-app-smoke for translated content, screenshot, device, and TestFlight proofs are explicitly NOT claimed.

## Residual Risk

- See handoff JSON `residual_risks` for the full list. Top three:
  1. LLM drafts are not native-reviewed; subtle wording errors are very possible.
  2. Argument-order assumptions inside German sentences may be wrong even though the format specifiers compile fine.
  3. The `needs-layout-check` status is mechanical, not earned; a downstream native reviewer must actually verify each entry.
