# owlory-ui-device-proof

Physical-device proof for selected Owlory Continue smoke paths. Captured manually on Raell Dottin's iPhone (device id `00008130-000A090910C1401C`, iOS 26.4.2) on commit `c49e5adb01c48ca3eae5ab96aedd09ae06a29c30` from a clean working tree on branch `claude/trusting-jepsen-0acf20`.

This pack is the device-verified mirror of [`owlory-ui-smoke-proof/`](../owlory-ui-smoke-proof/) for the paths that the device's live data could exercise honestly. Unlike the simulator pack, this is *not* generated by `make ui-smoke-proof`; iOS Release-runtime debug-only seed flags do not run on a physical device install. Recapture is a manual coordination loop, documented at the bottom of this file.

## Manifest integrity

`manifest.json` records `source_commit`, `captured_at`, the on-device build identifier, and per-PNG `name` / `file` / `sha256` / `size_bytes`. Do not hand-edit screenshots or the manifest; recapture if the surface changes and re-commit. A future commit that touches Continue, routing, or any surface in this pack must either update the screenshots (recapture against the new commit) or add a residual-risk note in its handoff explaining why the pack does not need refresh.

## Recapture procedure

This pack is not refreshed by a Makefile target because debug-only seed flags do not run on device installs. The deterministic flow is:

1. From a clean working tree at a committed SHA, build for device:
   ```bash
   xcodebuild build \
     -project owlory_xcode/Owlory.xcodeproj \
     -scheme Owlory \
     -configuration Debug \
     -destination 'platform=iOS,id=<DEVICE_UDID>' \
     -derivedDataPath /tmp/owlory-device-proof/DerivedData \
     -allowProvisioningUpdates
   ```
2. Install:
   ```bash
   xcrun devicectl device install app --device <DEVICE_UDID> \
     /tmp/owlory-device-proof/DerivedData/Build/Products/Debug-iphoneos/Owlory.app
   ```
3. Force-quit and relaunch Owlory on the device. Open Today → Build Info and screenshot first. Refuse to proceed if Build Info shows `no-git` or a mismatched commit/branch.
4. Capture additional surfaces only for what the device's live data naturally supports. Do not synthesize data on device for proof — record gaps honestly in the table above instead.
5. Save screenshots into this directory with the established `NN-kebab-case.png` names, regenerate `manifest.json` (sha256 hashes), and update the README's Build Info gate block, file table, and not-attempted-yet table to match the new run.
