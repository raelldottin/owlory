# App Localization HIG Multisurface Screenshot Harness

This directory is the proof output target for
`automation/smoke/capture_localized_surfaces.py --capture`. It is empty until a
real screenshot capture run lands proof artifacts here.

## What this is

A scoped HIG screenshot proof bundle. When populated by a successful capture
run, it contains:

- One PNG per `(locale, surface)` pair, named
  `NN-locale-<locale>-surface-<surface_id>.png`.
- `manifest.json` with per-capture `locale`, `surface_id`, `surface_label`,
  `file`, `bytes`, `sha256`, `navigation_steps_executed`,
  plus run-level `git_commit_short`, `target.udid`, `target.bundle_id`,
  `timestamp`, `proof_level`, and `non_claims`.
- This `README.md` (regenerated by the harness on each successful capture run).

## What this is not

- It is not a proof artifact today. As of 2026-05-18 no `--capture` run has
  landed; this directory exists so the harness has a stable, repo-managed
  output target and so the all-locale HIG evidence matrix
  (`automation/proofs/app-localization-hig-ui-matrix/manifest.json`) can
  reference it via `proof_references` entries.
- It does not claim translation quality.
- It does not claim full layout correctness.
- It does not claim device or TestFlight proof.
- It does not claim `hig-ui-reviewed` for any locale by itself; per-bucket HIG
  gate slices consume these proofs and feed the matrix.

## How to capture

```bash
make localization-multisurface-screenshot-idb-check
python3 automation/smoke/capture_localized_surfaces.py --list-surfaces
python3 automation/smoke/capture_localized_surfaces.py \
    --dry-run \
    --locales en de --surfaces today build-info
python3 automation/smoke/capture_localized_surfaces.py \
    --capture --udid <simulator-udid> \
    --locales en de --surfaces today build-info
```

The directory must be empty before `--capture`. To recapture, deliberately
clear the stale attempt first; the harness refuses to mix old and new
screenshots.

## Surface catalog

Run `python3 automation/smoke/capture_localized_surfaces.py --list-surfaces`
for the live catalog and per-surface `navigation_steps` plus
`settled_assertion_labels`. The catalog covers:

- `today` — settled Today launch surface
- `root-tab-train`, `root-tab-write`, `root-tab-career`, `root-tab-home` —
  root tabs
- `build-info` — Build Info screen with version/build/commit/branch fields
- `empty-state-today` — Today empty-state copy (requires a launch argument or
  fixture that starts the app with no Focus Three entries)
- `date-count-plural-today` — Today surface with at least one plural-formatted
  count and one locale-aware date string visible

Locale-specific settled labels can be added via
`--label-overrides path/to/labels.json` for languages where the English
fallback label does not match the localized accessibility label.
